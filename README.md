# DomainForge

A production-style .NET reference project focused on building maintainable, testable, and scalable applications using modern software engineering practices.

DomainForge is not a simple CRUD sample. It is designed as a practical playground for exploring architectural decisions, domain modeling, persistence strategies, and end-to-end testing approaches used in real-world systems.

---

## 🎯 Purpose

The goal of DomainForge is to create a reference implementation that demonstrates:

- Domain-Driven Design (DDD) principles
- Clean architecture concepts
- Separation of business rules from infrastructure concerns
- Entity Framework Core usage in a production-oriented way
- Acceptance testing using the real application composition root
- Test environments that reuse the application's actual configuration
- Incremental evolution of a software system using Agile practices

---

## 🏗 Architecture Overview

DomainForge follows a layered architecture inspired by Clean Architecture and DDD.
